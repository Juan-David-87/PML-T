Dataset Split

Data Preparation

The raw dataset contains 77,163 records across multiple years and sub-indices. We filtered for VIGENCIA 2021 (the most recent complete year) and pivoted so each entity has one row with its score per sub-index.

Sub-indices present in fewer than 50% of entities were excluded to avoid imputation bias. The final dataset was split 80/20 (train/test) for stability validation. DBSCAN is fully unsupervised — the split is used only to verify cluster consistency.

    DBSCAN Key Concepts
    • ε (epsilon): Maximum distance between two points for neighbourhood.
    • min_samples: Minimum points to form a dense core region.
    • Core points: Have ≥ min_samples within ε distance.
    • Border points: Reachable from a core point but not core themselves.
    • Noise points: Not reachable from any core point.

    Interpretation Guide
    Silhouette ≈ 0.35

    Scores between 0.25–0.50 indicate a reasonable structure. The main cluster is dense and well separated from the 5 critical outlier entities. Values close to 0 would indicate overlapping clusters.

    Davies-Bouldin ≈ 0.59

    Values below 1.0 are considered good. A DBI of ~0.59 confirms that the clusters are well separated relative to their intra-cluster scatter.

    Calinski-Harabasz ≈ 19

    CHI reflects the ratio of between-cluster to within-cluster variance. Given the natural density of this dataset (government entities with similar scores), this value is expected to be moderate.

    DBSCAN does not optimise cluster count — it discovers structure naturally from density.

    Risk Assignment Methodology

    Clusters are ranked by their mean Gobierno Digital composite score. The lowest-scoring cluster is labelled Alto risk, the middle Medio, and the highest Bajo. Noise entities (DBSCAN outliers) are marked Indeterminado pending expert review.
