Dataset Split

Data Preparation

The raw dataset contains records across multiple years and sub-indices. We filtered for VIGENCIA 2021 (the most recent complete year) and pivoted so each municipality has one row with its average score per sub-index.

Sub-indices present in fewer than 50 % of municipalities were excluded to avoid imputation bias. The final dataset was split 80/20 (train/test) to verify cluster stability. K-Means learns from the training set; the test set confirms that cluster assignments remain consistent.

    K-Means Key Concepts
    • k: Number of clusters to find — the main hyperparameter.
    • Centroid: The mean point of all observations in a cluster.
    • Inertia: Sum of squared distances from each point to its centroid.
    • n_init: Times the algorithm runs with different centroid seeds; best result is kept.
    • StandardScaler: Applied before K-Means so no sub-index dominates due to scale differences.

    Interpretation Guide
    Silhouette 0.25 – 0.50

    Scores in this range indicate a reasonable structure. Values close to 0 would indicate overlapping clusters; values near 1 indicate perfectly separated clusters. Government entity scores naturally overlap, so moderate silhouette values are expected.

    Davies-Bouldin < 1.0

    Values below 1.0 confirm that the clusters are well separated relative to their intra-cluster scatter. Lower is always better.

    Calinski-Harabasz

    Reflects the ratio of between-cluster to within-cluster variance. Higher values indicate denser, more separated clusters. Useful for comparing different k values on the same dataset.

    K-Means assigns every point to a cluster — there are no noise points. Use the silhouette plot to detect borderline municipalities.

    Risk Assignment Methodology

    Clusters are ranked by their mean Gobierno Digital composite score (average of all sub-index scores). The lowest-scoring cluster is labelled Alto risk, the next Medio-Alto, then Medio-Bajo, and the highest Bajo. Unlike DBSCAN, K-Means guarantees that every municipality is assigned to a cluster — no entity is left Indeterminado. This methodology aligns with CRISP-ML's requirement to link model outputs to actionable business decisions — supporting MinTIC's telecommunications infrastructure investment planning.
